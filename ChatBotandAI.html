Enhancing User Experience with AI Chatbots

Having worked with 3 projects involving Chat Bots, I have Consolidated my design methods in this case study

01 CONCEPT | BRAINSTORM | INFORMATION GATHERING

A chatbot is a software application that simulates conversation with human users through text or voice commands. Chatbots are typically domain-specific, meaning they are designed to assist users with a limited range of tasks within a particular context.

Project 1:
Celgene
Streamlined Order Tracking

Client: Celgene, a pharmaceutical company
Objective: Enhance customer experience with a user-friendly chatbot for order status and details retrieval.
UX Challenges:

  • Clarity and Ease of Use: Cater to users with varying technical backgrounds.
  • Efficiency: Minimize steps to retrieve order information.
  • Information Architecture: Order details need a clear and user-friendly format.
Design Solutions:
  • Simple Interface: Clean and concise interface with intuitive prompts and buttons.
  • Natural Language Processing (NLP): Users inquire about orders using natural language ("What's the status of my order?").
  • Order Summary: Well-organized layout displaying order details (product name, quantity, estimated delivery date).
Outcomes:
  • Increased customer satisfaction with a 24/7 order tracking channel.
  • Reduced call center inquiries regarding order status, freeing up resources.

Project 2:
Accenture Library
Empowering Employee Self-Service

Client:Accenture Library
Objective: Design a chatbot to assist employees with library-related tasks (employee details, borrowing history, project information).
UX Challenges:

  • Role-Based Access Control: Ensure relevant information based on employee role.
  • Multitasking: Enable conducting multiple tasks within a single chatbot session.
  • Security: Implement secure measures to protect sensitive employee data.
Design Solutions:
  • Role-Based Responses: Tailor chatbot responses and functionalities based on user roles (e.g., project details only for authorized personnel).
  • Multi-Step Workflows: Guide users through multi-step tasks seamlessly, allowing switching between inquiries within a session.
  • Secure Data Access: Integrate secure authentication methods (e.g., employee ID verification) for accessing sensitive data.
Outcomes:
  • Increased employee self-service, reducing reliance on library staff for routine inquiries.
  • Improved accessibility to library resources 24/7, enhancing employee productivity.
  • Reduced administrative workload for library staff, freeing up time for strategic tasks.

Project 3:
CommonFloor
Simplified Rental Property Search

Client: CommonFloor, a real estate platform
Objective: Design a chatbot to assist users in searching for rental properties within their desired locality.
UX Challenges:

  • Understanding User Intent: Effectively interpret user queries to identify specific search criteria (location, budget, amenities).
  • Search Refinement: Allow users to refine search parameters based on chatbot suggestions.
  • Visualization: Present property listings in an engaging and informative way within a text-based interface.
Design Solutions:
  • Conversational Search: Utilize natural language processing to understand user queries' context and intent.
  • Iterative Refinement: Implement a mechanism for users to adjust search criteria based on chatbot recommendations.
  • Rich Text Formatting: Use text formatting techniques to highlight key property details (rent amount, location) within the chat window.
